Reto 4. Creación de usuarios y gestión de permisos políticas y directivas.

1. Elección de topologías de red

A la hora de implementar la red para el plano del proyecto, después de informarnos sobre las ventajas y desventajas de cada una, hemos escogido para la infraestructura las siguientes topologías, que es la mixta, en la que estará compuesta por:

Usuario base Labdoo

Definición:

En este apartado tuvimos que crear un usuario base que se llama Labdoo, que es como se llama el cliente que usamos para realizar la configuración.

 

Estructura de carpeta compartidas

Definición:

En este apartado tuvimos que crear una estructura de carpetas en el servidor que posteriormente compartimos con el usuario base con permisos de escritura y lectura.

Definición de las políticas de gestión de la escuela

Por último en este apartado explicamos el porqué de cada permiso y normativa:

Fondo de escritorio: Deshabilitamos el cambio de fondo de escritorio para que los usuarios no puedan cambiarlo y añadir cualquier imagen.

Agregar y quitar elementos: Deshabilitamos la opción de que se puedan añadir o eliminar los elementos del escritorio para que no añadan cosas irrelevantes y eliminen las útiles.

Windows Update: Habilitamos la opción de que no aparezca el apartado de Windows Update en configuración para que no puedan cambiar la versión del sistema operativo y así todos usen la misma.

Carpetas compartidas: Deshabilitamos esta opción para que los usuarios no puedan crear carpetas compartidas ya que ya tienen una creada, que es la que hemos creado anteriormente.

Escritorio remoto: Deshabilitamos la opción del uso del escritorio remoto para que los usuarios no puedan utilizarlo ya que no es una opción que tengan que utilizar.

Uso compartido de red: Habilitamos la opción de impedir que los usuarios compartan archivos ya que así no podrán pasarse los trabajos.

Ubicación y sensores: Desactivamos el cambio de ubicación y sensores ya que es una acción a la cual no necesitan tener acceso.
